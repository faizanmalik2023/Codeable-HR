# CodeableHR вЂ” HR Dashboard Spec (Gospel for Web Build)

> Fully self-contained. Carries everything needed to rebuild the HR experience as a Next.js web dashboard. Shared sections (В§2вЂ“В§5) are identical across the Employee/HR/Admin specs; the role sections (В§1, В§6вЂ“В§10) are HR-specific. HR includes all personal self-service (apply leave, submit EOD/claims, view own salary) **plus** org management powers below.

---

## 0. How to read this doc
- **Wire value** = exact string sent/received for an enum; always use it in requests.
- **Label** = display text.
- API paths are the app's endpoint constants relative to `{BASE_URL}/{API_VERSION}` (see В§3).
- "Editable vs read-only", "dialog vs sheet vs page" are called out because they drive web layout (modal vs inline vs route).
- **Mobile в†’ Web**: mobile bottom-sheets/full-screen pushes become right-side drawers / modals / routes on web (noted per screen).

## 1. Role Overview вЂ” HR

**Landing:** `/hr-home` (Dashboard). **Nav:** left drawer, 12 destinations + Settings + Logout.

**HR CAN do (superset of Employee self-service + management):**
- Everything an Employee does for themselves: dashboard, apply leave, submit/manage own EOD, submit insurance/expense claims, view own salary/payslips, raise HR tickets, notifications, profile, policies (read).
- **Leaves:** review & approve/reject org leave **requests**; see who's **on leave today**.
- **EOD:** view/manage EOD reports (own + team), delete.
- **Attendance:** view all employees' attendance logs, drill into per-employee month.
- **People:** browse all employees; view & **edit** employee master records; **add** employees; **promote/increment/deactivate**; manage perks/documents.
- **Departments:** create/edit/delete; manage members & manager.
- **Projects:** full projects management (create/edit/archive; members, tasks, milestones, documents, analytics).
- **Payroll:** generate/stage, review, **release** payslips, amend individual slips, download.
- **Insurance & Expense claims:** approval queues + decisions.
- **HR Help:** triage employee tickets, reply, change status/resolve.
- **Announcements:** broadcast to all / department / role.
- **Policies:** (HR can create/edit/delete вЂ” same shared PoliciesScreen with management affordances).

**HR CANNOT do:** finance/treasury, equity/distributions, project income, loans, biometric devices, holiday creation is admin-driven (HR consumes), and admin-only org-wide finance. (These live in the Admin spec.)

## 2. Auth & Session (shared core)
### 2.1 Login & onboarding
- Public routes: `/`, `/login`, `/forgot-password`, `/otp`, `/new-password`. Unauthed non-public в†’ `/login`.
- Post-login role landing: `admin в†’ /admin-home`, `hr в†’ /hr-home`, `employee|null в†’ /employee-home`.
### 2.2 Auth endpoints (no bearer)
`POST /auth/login {email,password}` В· `POST /auth/google {id_token}` В· `POST /auth/refresh-token {refresh_token}` (rotates) В· `POST /auth/logout` В· `POST /auth/forgot-password {email}` В· `POST /auth/verify-otp {email,otp}в†’{reset_grant}` В· `POST /auth/reset-password {reset_grant,new_password}` В· `POST /auth/password/change {current_password,new_password}`. All return the standard envelope; login/google return `{ user, tokens:{token,refresh_token} }`.
### 2.3 Token handling
Bearer on every non-auth request. On 401 (non-auth): single-flight refresh в†’ retry once; on refresh failure в†’ force logout + clear + `/login`.
### 2.4 Password policy
min 8, one uppercase, one number, one special char (live checklist).

## 3. Global Conventions (shared core)
### 3.1 Environments
dev/staging/prod flavors; each `.env` has `BASE_URL`, `API_VERSION`, `GOOGLE_SERVER_CLIENT_ID`; requests в†’ `{BASE_URL}/{API_VERSION}/...`.
### 3.2 HTTP
Dio; JSON; 60s timeouts; GET/POST/PUT/PATCH/DELETE + multipart for uploads.
### 3.3 Response envelope
Success `{ statusCode, data, error:null }`; error `{ statusCode, data:null, error:{ code, message, timestamp, details[] } }`. Fallback messages: 400/401/403/404/429/500 as in Employee spec В§3.3.
### 3.4 Pagination
`PaginationModel { currentPage, totalPages, totalItems }`, `hasMore=currentPage<totalPages`; tolerant of snake_case + legacy `{page,limit,total}`. Requests send `page`(1-idx)+`limit`. List responses `{ items, pagination, counts }` where counts power whole-set filter badges.
### 3.5 Money
Comma-group input; PKR default (finance allows USD); display `5,600.50`; compact `100k/5.6M/3T`.
### 3.6 Dates
Wire date `YYYY-MM-DD`, month key `YYYY-MM`, ISO datetime; ordinal display + `time_ago`.
### 3.7 Standard UI states
Loading skeleton / load-more spinner / empty (message per screen) / error+retry (or toast over cache) / pull-to-refresh / success toast вЂ” apply to every list & detail.
### 3.8 Endpoint reference (HR-relevant)
Dashboard `GET /dashboard`. Leaves: `/leaves/quota|types|apply|history`, `GET /leaves/requests`, `GET /leaves/on-leave-today`, `PATCH /leaves/{id}/decision`, `/leaves/team[/{id}]`. EOD: `/eods`, `/eods/submit`, `/eods/{id}`, `DELETE /eods/{id}`, `/eods/team*`. Attendance: `GET /attendance/employees`, `GET /attendance/employees/{code}/logs?month&year`. Employees: `GET /employees`, `GET /employees/{id}`, `GET /employees/designations`, `POST /employees`, `PATCH /employees/{id}`, `POST /employees/{id}/promote|increment|deactivate`, `GET /salary/employee/{userId}`, `PUT /salary/employee/{userId}`. Projects: `GET /projects`, `GET /projects/options`, `GET/POST/PATCH/DELETE /projects[/{id}]`, `.../members`, `.../tasks`, `.../milestones`, `.../documents`, `.../eods`, `.../analytics`. ## 8. Cross-feature flows вЂ” HR
1. **Leave approval**: employee applies в†’ appears in `/hr-leave-requests` в†’ approve/reject в†’ employee notified + activity + balance.
2. **Onboarding**: Add Employee в†’ record created (employee_code) в†’ appears in People в†’ salary set в†’ payroll picks it up.
3. **Payroll cycle**: generate (stage pending) в†’ review/amend slips в†’ release all (emails + notifies) в†’ employees see payslips.
4. **Claim decision**: employee submits в†’ `/hr-*-claims` queue в†’ approve/reject (reason) в†’ notified.
5. **HR Help triage**: employee raises в†’ `/hr-hr-help` в†’ reply thread в†’ resolve в†’ reply disabled.
6. **Announcement**: compose в†’ target all/department/role в†’ recipients notified.
7. **Project delivery**: create project в†’ add members в†’ tasks/milestones в†’ members' EODs feed activity/analytics.
